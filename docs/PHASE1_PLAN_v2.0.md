## 🤝 前后端协同流程

### 1. API 接口定义（开发前）

**负责人：** 架构设计虾 + 后端开发虾 + 前端开发虾

**流程：**
1. 后端定义 API 接口（路径、方法、请求/响应）
2. 前端确认接口格式
3. 写入接口文档（Swagger）
4. 三方确认

**示例：**
```yaml
POST /api/search
请求：
  theme: string
响应：
  code: number
  message: string
  data:
    search_id: number
    weibo_hot_search: array
    zhihu_hot_list: array
```

---

### 2. Mock 数据（开发中）

**负责人：** 前端开发虾

**流程：**
1. 前端根据 API 定义创建 Mock 数据
2. 前端使用 Mock 数据开发 UI
3. 后端开发真实 API
4. 联调时替换 Mock 数据

**示例：**
```javascript
// frontend/src/mock/search.js
export const mockSearchResponse = {
  code: 200,
  message: "success",
  data: {
    search_id: 1,
    weibo_hot_search: [...],
    zhihu_hot_list: [...]
  }
}
```

---

### 3. 联调测试（开发后）

**负责人：** 后端开发虾 + 前端开发虾 + 测试运维虾

**流程：**
1. 后端部署 API 到测试环境
2. 前端配置 API 地址
3. 联调测试
4. 修复问题
5. 测试验收

## ⚠️ 研发注意事项

### 1. 跨平台适配（❗❗❗）

**必须遵守：**
- ✅ 使用 `pathlib.Path`
- ✅ 文件编码 `UTF-8`
- ✅ 命令行 `sys.executable`
- ✅ 配置文件英文注释

**严格禁止：**
- ❌ 硬编码路径
- ❌ 默认编码
- ❌ `python3` 命令
- ❌ 中文注释（配置文件）

---

### 2. API 接口规范

**请求格式：**
```json
{
  "theme": "AI 效率工具"
}
```

**响应格式：**
```json
{
  "code": 200,
  "message": "success",
  "data": {...}
}
```

**错误处理：**
```json
{
  "code": 500,
  "message": "Search failed",
  "error": "Detailed error message"
}
```
