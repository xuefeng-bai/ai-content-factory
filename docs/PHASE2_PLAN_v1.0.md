**API 接口：**
```
GET    /api/prompts              - 获取 Prompt 列表
GET    /api/prompts/:id          - 获取 Prompt 详情
POST   /api/prompts              - 创建 Prompt
PUT    /api/prompts/:id          - 更新 Prompt
DELETE /api/prompts/:id          - 删除 Prompt
POST   /api/prompts/:id/versions - 创建新版本
GET    /api/prompts/:id/versions - 获取版本历史
POST   /api/prompts/:id/versions/:versionId/publish - 发布版本
POST   /api/prompts/test         - 测试 Prompt 效果
```

## ⚠️ 研发注意事项

### 1. Prompt 管理

**必须遵守：**
- ✅ 系统内置 Prompt 标记为 `is_system=1`
- ✅ 系统内置 Prompt 不可删除（可禁用）
- ✅ 每次修改 Prompt 创建新版本
- ✅ 发布前必须测试 Prompt 效果

**严格禁止：**
- ❌ 直接修改生产环境 Prompt
- ❌ 删除有历史数据的 Prompt
- ❌ 不测试就发布 Prompt

---

### 2. 数据库操作

**必须遵守：**
- ✅ 使用参数化查询（防止 SQL 注入）
- ✅ 事务操作（保证数据一致性）
- ✅ 外键约束（保证数据完整性）

**严格禁止：**
- ❌ 拼接 SQL 字符串
- ❌ 跳过事务直接操作
- ❌ 忽略外键约束

---

### 3. AI 服务调用

**必须遵守：**
- ✅ 设置超时时间（文本 30s，图片 60s）
- ✅ 添加重试机制（最多 3 次）
- ✅ 记录调用日志（用于分析优化）

**严格禁止：**
- ❌ 无超时调用
- ❌ 不处理异常
- ❌ 不记录日志

---

### 4. 跨平台适配

**必须遵守：**
- ✅ 使用 `pathlib.Path`
- ✅ 文件编码 `UTF-8`
- ✅ 命令行 `sys.executable`
- ✅ 配置文件英文注释

**严格禁止：**
- ❌ 硬编码路径
- ❌ 默认编码
- ❌ `python3` 命令
- ❌ 中文注释（配置文件）
