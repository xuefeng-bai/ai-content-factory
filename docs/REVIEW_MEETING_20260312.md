## 🎤 需求介绍（已完成）

**核心工作流：**
```
选择模板 → 输入主题 → AI 联网搜索 → 推荐选题 → 选择选题 → 生成内容 → 保存到历史
```

**核心功能：**
- 5 大搜索渠道（微博/知乎/抖音/小红书/X）
- 3-5 个选题推荐
- 3 平台内容生成（抖音/公众号/小红书）
- AI 配图生成（1-5 张）
- 历史内容管理

## 🙋 各角色提问和风险预判

### 🦐🏗️ 架构设计虾（技术架构）

#### 问题 1：5 大搜索渠道的技术实现
**问题描述：**
- 微博热搜、知乎热榜、抖音/快手、小红书、X（Twitter）
- 这些平台的 API 开放程度不同，如何获取数据？

**风险评估：** 🔴 高风险

**可能方案：**
| 平台 | 方案 | 难度 | 稳定性 |
|------|------|------|--------|
| 微博热搜 | 爬虫/第三方 API | 🟡 中 | 🟡 中 |
| 知乎热榜 | 爬虫/第三方 API | 🟡 中 | 🟡 中 |
| 抖音/快手 | 爬虫（难度大） | 🔴 高 | 🔴 低 |
| 小红书 | 爬虫（反爬严） | 🔴 高 | 🔴 低 |
| X（Twitter） | 官方 API（收费） | 🟢 低 | 🟢 高 |

**建议：**
- MVP 阶段先用 2-3 个稳定渠道（微博/知乎/X）
- 抖音/小红书后期通过 RPA 或第三方服务

---

#### 问题 2：AI 配图的并发和成本
**问题描述：**
- 公众号 1-5 张图 + 小红书封面 1 张 = 最多 6 张图/次
- DashScope 文生图 API 有并发限制和成本

**风险评估：** 🟡 中风险

**计算：**
```
假设每天生成 10 次内容
每次平均 4 张图 = 40 张图/天
DashScope 文生图约 0.12 元/张
每天成本：40 × 0.12 = 4.8 元
每月成本：4.8 × 30 = 144 元
```

**建议：**
- 限制每次生成的最大图片数量（如最多 3 张）
- 提供"跳过配图"选项
- 图片生成异步处理，不阻塞主流程

---

#### 问题 3：模板配置的存储结构
**问题描述：**
- 模板配置用 JSON 存储，但查询和筛选可能不方便
- 是否需要独立的模板表？

**风险评估：** 🟢 低风险

**建议：**
- MVP 阶段用 JSON（快速迭代）
- 后期如果模板功能复杂，再拆独立表

---

### 🦐💻 后端开发虾（后端开发）

#### 问题 4：AI 搜索的实时性
**问题描述：**
- AI 联网搜索需要调用 web_search 工具
- 搜索 5 大平台可能需要多次 API 调用
- 响应时间可能较长（10-30 秒）

**风险评估：** 🟡 中风险

**建议：**
- 前端增加进度提示（"正在搜索微博热搜...正在搜索知乎热榜..."）
- 后端异步处理，WebSocket 推送进度
- 设置超时时间（如 60 秒），超时返回部分结果

---

#### 问题 5：选题推荐的算法逻辑
**问题描述：**
- 如何从搜索结果中提炼出 3-5 个选题？
- 需要明确的推荐逻辑，否则 AI 生成的选题可能质量不稳定

**风险评估：** 🟡 中风险

**建议：**
- 定义选题推荐的结构化 Prompt
- 要求 AI 输出固定格式（类型、优势、推荐度）
- 增加人工评分反馈，优化推荐算法

---

#### 问题 6：内容生成的长度控制
**问题描述：**
- 抖音口播文案：30 秒/60 秒/90 秒，字数如何控制？
- 公众号文章：1500 字/3000 字/5000 字，AI 可能不准确

**风险评估：** 🟢 低风险

**建议：**
- Prompt 中明确字数范围（如"500-600 字"）
- 生成后校验字数，超出则截断或重新生成
- 前端显示预估字数/时长

---

### 🦐🎨 前端开发虾（前端开发）

#### 问题 7：模板选择的交互设计
**问题描述：**
- 首页需要展示 3 套预设模板 + 自定义配置
- 如何清晰展示每个模板的特点？

**风险评估：** 🟢 低风险

**建议：**
- 卡片式展示，每个模板显示关键信息
- 鼠标悬停显示详细说明
- 自定义配置用弹窗，避免页面跳转

---

#### 问题 8：内容预览页的复杂度
**问题描述：**
- 需要同时展示三平台内容（抖音 + 公众号 + 小红书）
- 公众号文章可能很长，配图也需要展示
- 页面可能过长，用户体验不好

**风险评估：** 🟡 中风险

**建议：**
- 标签页切换展示（抖音 | 公众号 | 小红书）
- 公众号文章支持折叠/展开
- 配图用缩略图，点击放大

---

#### 问题 9：一键复制的实现
**问题描述：**
- 抖音文案：纯文字，容易复制
- 公众号文章：Markdown + 图片 URL，如何复制？
- 小红书笔记：文案 + 封面图，如何复制？

**风险评估：** 🟡 中风险

**建议：**
- 抖音：一键复制文字
- 公众号：提供"复制 Markdown"和"下载 Markdown 文件"
- 小红书：文案复制 + 图片单独下载
- 增加"打包下载"功能（文案 + 所有图片）

---

### 🦐🔧 测试运维虾（测试运维）

#### 问题 10：AI 生成内容的质量稳定性
**问题描述：**
- AI 生成的内容质量可能不稳定
- 如何保证生成的文案可用？

**风险评估：** 🔴 高风险

**建议：**
- 定义内容质量检查清单
- 增加人工审核环节（MVP 阶段必须）
- 收集用户反馈，持续优化 Prompt

---

#### 问题 11：图片生成的合规性
**问题描述：**
- AI 生成的图片可能涉及版权问题
- 公众号和小红书商用是否有风险？

**风险评估：** 🔴 高风险

**建议：**
- 查阅 DashScope 的使用条款
- 增加版权声明
- 提供商用图片库选项（如 Unsplash）

---

#### 问题 12：历史内容的存储成本
**问题描述：**
- 每次生成的内容都保存到数据库
- 图片 URL 可能过期，是否需要下载存储？
- 长期积累后数据库大小问题

**风险评估：** 🟡 中风险

**建议：**
- MVP 阶段只存 URL（快速上线）
- 后期考虑图片下载存储
- 增加历史记录清理功能（如保留最近 100 条）
