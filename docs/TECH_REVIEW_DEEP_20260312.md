### 模块 1：AI 联网搜索

#### 技术选型

| 选项 | 方案 | 优缺点 | 最终选择 |
|------|------|--------|---------|
| **微博热搜** | 直接爬虫 | ✅ 免费 ✅ 实时 ❌ 反爬 | ✅ 选定 |
| | 第三方 API | ✅ 稳定 ❌ 收费 (50 元/月) | ❌ 不选 |
| | web_search | ✅ 简单 ❌ 数据不结构化 | ❌ 备选 |

| **知乎热榜** | 直接爬虫 | ✅ 免费 ✅ 实时 ❌ 需登录 | ✅ 选定 |
| | 第三方 API | ✅ 稳定 ❌ 收费 | ❌ 不选 |
| | web_search | ✅ 简单 ❌ 数据不结构化 | ❌ 备选 |

#### 深度技术难点分析

**微博爬虫难点：**
1. 🔴 **反爬机制** - IP 限流、验证码
   - 对策：请求间隔 3 秒 + User-Agent 轮换 + 代理 IP 池
   - 风险等级：🟡 中（可控）

2. 🟡 **HTML 结构变化** - 微博可能改版
   - 对策：XPath 选择器 + 定期校验
   - 风险等级：🟢 低（可维护）

3. 🟡 **数据清洗** - 热搜榜含广告
   - 对策：过滤"广告"标签
   - 风险等级：🟢 低（简单）

**知乎爬虫难点：**
1. 🔴 **登录验证** - 需 Cookie 才能访问
   - 对策：手动登录一次 → 保存 Cookie → 定期刷新
   - 风险等级：🟡 中（需维护）

2. 🟡 **反爬机制** - 请求频率限制
   - 对策：请求间隔 5 秒 + Cookie 轮换
   - 风险等级：🟡 中（可控）

3. 🟡 **数据解析** - 热榜数据结构复杂
   - 对策：正则 + JSON 解析结合
   - 风险等级：🟢 低（可解决）

#### 备选方案（如果爬虫失败）

**方案 A：web_search 工具**
- 搜索"微博热搜 2026"和"知乎热榜"
- 优点：简单快速
- 缺点：数据不够结构化，实时性差

**方案 B：用户手动输入**
- 提供输入框，用户手动粘贴热搜榜
- 优点：100% 稳定
- 缺点：用户体验差
